## 简短的历史回顾：

在九十年代早期，所有数据都由服务端处理验证，而当时带宽很低，如果提交数据有错，经过服务端验证后再返回再次修改再上传的时间花销是巨大的，因此在用户本地的客户端也分担数据的部分验证工作显得很必要， JavaScript 就是基于这种背景诞生的。一开始的计划是在客户端和服务端都同步使用它。

此时的 JS 还没有统一标准，微软为了推广自己的IE3，也设计了自己名为 JScript 的 JS 实现。为弥合分野，在九十年代末期，ECMA制定了ECMA-262标准。

> 自此以后，各家浏览器均以ECMAScript作为自己JavaScript实现的依据，虽然具体实现的程度各有不同。
>
> 也是基于这种背景，JS可以说并不是一门编程语言，而是由API集合支撑的一系列 ES 实现。

## JavaScript实现：

完整的JavaScript实现包含以下几个不同的部分，在浏览器中，每个HTML文档都对应一个window对象和一个document对象：

· 核心（ECMAScript，即ECMA-262标准）

· DOM（Document Object Model，文档对象模型）

· BOM（Browser Object Model，浏览器对象模型）

![image-20260415141244366](/assets/blog_res/2026-04-15-read1.assets/image-20260415141244366.png)

### ECMAScript：

在基本的层面，ECMAScript描述这门语言的如下部分：语法、类型、语句、关键字、保留字、操作符和全局对象。ECMAScript只是对实现这个规范描述的所有这些部分的一门语言的称呼，而JavaScript实现了ECMAScript。

![image-20260415141633451](/assets/blog_res/2026-04-15-read1.assets/image-20260415141633451.png)

### DOM：

DOM（Document Object Model，文档对象模型）是XML的一套API（Application Programming Interface，应用编程接口），经过扩展可以在HTML中使用。DOM将整个页面抽象为一组分层节点。HTML或XML页面的每个组成部分都是一种节点，包含不同的数据。比如下面的HTML页面：

```html
<html>
    <head>
        <title>Sample Page</title>
    </head>
    <body>
        <p> Hello World!</p>
    </body>
</html>
```

这些代码通过DOM可以表示为一组分层节点，使用DOM API，可以轻松地删除、添加、替换、修改节点。

![image-20260415142516373](/assets/blog_res/2026-04-15-read1.assets/image-20260415142516373.png)

DOM标准同样也是弥合不同浏览器分野的产物，DHTML即html+css+javascript的集成。

![image-20260415142649440](/assets/blog_res/2026-04-15-read1.assets/image-20260415142649440.png)

### BOM：

IE3和Netscape Navigator 3最早推出了BOM（Browser Object Model，浏览器对象模型）API，用于支持访问和操作浏览器的窗口。使用BOM，开发者可以操控浏览器显示的页面之外的部分。而BOM真正独一无二的地方，当然也是问题最多的地方，就是它是唯一一个没有相关标准的JavaScript实现。HTML5的出现改变了这个局面，这个版本的HTML以正式规范的形式涵盖了尽可能多的BOM特性。由于HTML5的出现，之前很多与BOM有关的问题都迎刃而解了。

总的来说，BOM主要针对浏览器窗口和窗格(frame)，不过人们通常会把任何特定于浏览器的扩展都归在BOM的范畴内。

在js中，操作DOM的核心对象是document，操作BOM的核心对象是window，document是window的属性，DOM包含在BOM中。虽然BOM没有统一标准，但现代浏览器基本实现了一致的API。**框架开发中**，尽量使用框架提供的抽象，避免直接操作BOM/DOM。

```javascript
// DOM的核心对象是document
document.getElementById('myId');      // 通过ID获取元素
document.querySelector('.myClass');   // 通过选择器获取元素
document.createElement('div');        // 创建新元素
document.body.appendChild(newDiv);    // 添加元素到文档
```

```javascript
// BOM的核心对象是window
window.alert('提示信息');            // 弹出对话框
window.location.href = 'https://...'; // 跳转页面
window.localStorage.setItem('key', 'value'); // 本地存储
window.navigator.userAgent;          // 浏览器信息
```

## HTML中的JavaScript：

### \<script>元素：

将JavaScript插入HTML的主要方法是使用\<script>元素，它有下列8个属性。

![image-20260415145351759](/assets/blog_res/2026-04-15-read1.assets/image-20260415145351759.png)

使用\<script>的方式有两种：通过它直接在网页中嵌入JavaScript代码，以及通过它在网页中包含外部JavaScript文件。一般来说，在\<script>元素中的代码被解释完成之前（包括下载代码和执行代码两步），此元素下方的页面其余内容不会被加载，也不会被显示。

如果使用 async 属性，则在下载时不会阻塞html，但在下载完后会立刻执行，执行时会造成阻塞；如果使用 defer 属性，会在html解析完成后执行，则全程不会阻塞。一般来说，关键的脚本采用普通模式，非关键脚本可考虑采用异步形式。

> DOMContentLoaded 是浏览器提供的一个重要事件，表示初始的HTML文档已经完全加载和解析完成，不需要等待样式表、图像和子框架的加载完成，此时可以可以开始操作页面元素。
> load 代表所有资源（图片、CSS等）加载完成，会在 DOMContentLoaded 后执行。

```html
<script async src="script.js"></script>
<!--
  特点：
  1. 异步下载，不阻塞HTML解析
  2. 下载完成后立即执行，执行时会阻塞HTML解析
  3. 多个async脚本执行顺序不确定
  4. 不适合有依赖关系的脚本
-->
```

```html
<script defer src="script.js"></script>
<!--
  特点：
  1. 异步下载，不阻塞HTML解析
  2. 等到HTML解析完成后才执行（DOMContentLoaded之前）
  3. 多个defer脚本按顺序执行
  4. 适合有依赖关系的脚本
-->
```
